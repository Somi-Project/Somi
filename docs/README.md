# Somi Docs

This folder holds the durable written maps for Somi.

For basic users:
- Start with `architecture/NEWCOMER_CHECKLIST.md` for the fastest orientation.
- Use `architecture/CONTRIBUTOR_MAP.md` to find the main surfaces: chat,
  search, coding, memory, Telegram, and operations.
- Use `release/` when you need release notes, readiness artifacts, or merge
  prep context.

For developers:
- `architecture/` is the source of truth for how Somi is split across runtime,
  GUI, memory, search, coding, and integrations.
- `PLUGIN_FEDERATION.md` explains how outside `SKILL.md` ecosystems map into
  Somi's plugin network.
- `SYSTEM_MAP.md` is the best high-level graph when tracing a request end to
  end.

If you are new to the repo, read this folder before reverse engineering the
codebase from imports alone.
