# Communications and Power Continuity

When connectivity is weak, preserve the essentials first.

Communications:
- Keep one trusted message path for status, one for escalation, and one offline log for delayed delivery.
- Prefer short, structured updates with time, place, resource status, and next action.
- Cache vital docs locally so users can still retrieve repair, medical, and logistics notes without the web.

Power:
- Put critical loads on a written priority list before batteries run low.
- Charge devices in scheduled windows instead of ad hoc usage.
- Track fuel, charge cycles, and inverter heat so the team sees failures before they cascade.

Node operations:
- Keep one local copy of the latest support bundle, release gate, and important manuals.
- Record what data came from live web sources versus bundled local knowledge so stale facts are easier to spot later.
